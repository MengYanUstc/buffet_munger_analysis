# 管理层定性分析评分规则（Module 2）

**总分：10 分**

| 维度 | 类型 | 分值 | 数据来源 |
|------|------|------|----------|
| 资本配置能力 | AI_BASED | 6 | ROIC趋势(SQLite) + 股权质押 + 分红/并购(网络搜索) |
| 管理层诚信 | AI_BASED | 4 | 违规记录(网络搜索) + 减持行为(网络搜索) |

---

## 1. 资本配置能力（6 分）—— AI_BASED

### 1.1 定量基准分（代码计算）

从中性偏上 4.5 分开始，根据以下规则调整：

**ROIC 趋势调整**：
| ROIC 趋势 | 调整 |
|-----------|------|
| 明显下降 | -1.5 |
| 温和下降 | -0.5 |
| 明显上升 | +0.5 |
| 温和上升 | +0.5 |
| 基本稳定 | 0 |

**股权质押调整**：
| 质押比例 | 调整 |
|----------|------|
| > 50% | -1.5 |
| > 30% | -0.5 |
| <= 30% | 0 |

**base_score** = round(调整后分数 / 0.5) * 0.5，限制在 [0, 6]

### 1.2 AI 微调

LLM 在 `base_score` 基础上做 [-0.5, +0.5] 微调。

微调依据（通过联网搜索获取）：
- 分红政策：是否稳定持续？分红比例是否合理（30%-70%）？
- 并购与扩张：重大并购后业绩表现？商誉减值风险？
- 综合资本使用效率判断

**数据流**：
```
SQLite (roic) -> ManagementDataFetcher (分红/质押/并购/违规搜索)
  -> CapitalAllocationPlugin._compute_base_score()
  -> AiScoringEngine -> LLM 微调
```

## 2. 管理层诚信（4 分）—— AI_BASED

### 2.1 关键词扫描惩罚（代码计算）

**一票否决**：若 `violations.has_fraud_flag = True`，直接 0 分。

**关键词匹配惩罚**（在违规记录和持股变动文本中扫描）：

| 关键词模式 | 扣分 | 描述 |
|-----------|------|------|
| 证监会立案/立案调查/欺诈发行/财务造假/系统性腐败/终身市场禁入 | 4.0 | 一票否决级 |
| 行政处罚/监管函/通报批评/责令改正/内幕交易/操纵市场/违规披露 | 1.5 | 严重违规 |
| 关联交易.*利益输送/利益输送/资金占用/违规担保/实控人占用 | 1.0 | 治理缺陷 |
| 离婚式减持/清仓式减持/减持引发股价大跌/敏感期减持/高管集体减持 | 1.5 | 异常减持 |
| 大额减持/持续减持/减持套现/高管密集减持/减持计划 | 0.5 | 一般减持 |

**penalty_score** = max(0, 4.0 - 总扣分)

### 2.2 AI 微调

LLM 在 `penalty_score` 基础上做 [-0.5, +0.5] 微调。

微调依据（通过联网搜索获取）：
- 违规记录详情：是否已整改？是否重复违规？
- 减持行为合理性：是否有合理解释？
- 治理结构：董事长是否频繁更替？关联交易风险？

**数据流**：
```
ManagementDataFetcher (违规/减持搜索)
  -> IntegrityPlugin._compute_penalty_score()
  -> AiScoringEngine -> LLM 微调
```

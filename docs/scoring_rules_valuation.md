# 估值分析评分规则（Module 5）

**总分：20 分 = 定量 17 分 + 定性 3 分**

| 维度 | 类型 | 分值 | 数据来源 |
|------|------|------|----------|
| 绝对估值水平 | 定量 | 6 | SQLite `valuation_metrics.pe_ttm, pb, ps_ttm` |
| 相对估值 | 定量 | 4 | SQLite `pe_percentile_5y, pe_vs_industry` |
| 长期 PEG | 定量 | 3 | SQLite `pe_ttm + financial_reports.profit_cagr` |
| DCF 安全边际 | 定量 | 4 | SQLite `fcf + net_profit` -> DCF 模型 |
| 增长确定性 | 定性(Coze) | 3 | 复用 business_model 定性缓存 |

---

## 1. 绝对估值水平（6 分）—— 定量

### 1.1 PE 基础分（主要指标）

| PE | 得分 |
|----|------|
| < 12 | 6.0 |
| < 15 | 5.0 |
| < 20 | 4.0 |
| < 25 | 3.0 |
| < 30 | 2.0 |
| < 35 | 1.0 |
| >= 35 或 <= 0 或 > 100 | 0.0 |

### 1.2 PB 加分（辅助指标）

| PB | 加分 |
|----|------|
| < 2.0 | +1.0 |
| < 3.0 | +0.5 |
| < 4.0 | 0 |
| >= 4.0 | -0.5 |

### 1.3 PS 加分（辅助指标）

| PS | 加分 |
|----|------|
| < 2.0 | +1.0 |
| < 3.0 | +0.5 |
| < 4.0 | 0 |
| >= 4.0 | -0.5 |

### 1.4 最终分

**absolute_score** = max(0, min(6, PE基础分 + PB加分 + PS加分))

**数据流**：`SQLite valuation_metrics -> pe_ttm, pb, ps_ttm -> calculate_absolute_valuation_score()`

---

## 2. 相对估值（4 分）—— 定量

### 2.1 历史估值分位得分（0-3 分）

| PE 历史分位 | 得分 |
|-------------|------|
| < 20% | 3.0（极低，极度低估） |
| < 40% | 2.0（偏低） |
| < 60% | 1.5（中等） |
| < 80% | 0.5（偏高） |
| >= 80% | 0.0（极高，高估） |

### 2.2 相对行业平均得分（0-1 分）

`relative_ratio = 公司PE / 行业PE`

| relative_ratio | 得分 |
|----------------|------|
| <= 0.85 | 1.0（显著低于行业） |
| < 1.15 | 0.5（与行业接近） |
| >= 1.15 | 0.0（高于行业） |

### 2.3 一致性调整

- 历史和行业**同时低估**（分位 < 40% 且 ratio <= 0.85）：+0.5
- 历史和行业**同时高估**（分位 > 60% 且 ratio >= 1.15）：-0.5

### 2.4 最终分

**relative_score** = max(0, min(4, 历史分位得分 + 行业得分 + 一致性调整))

**数据流**：`SQLite valuation_metrics -> pe_percentile_5y, pe_vs_industry -> calculate_relative_valuation_score()`

---

## 3. 长期 PEG（3 分）—— 定量

### 3.1 PEG 计算

`PEG = PE / (profit_cagr * 100)`

profit_cagr 为近5年净利润复合增长率（小数形式，如 0.15 表示 15%）。

### 3.2 评分规则

| PEG | 得分 |
|-----|------|
| <= 0 | 0.0 |
| <= 0.8 | 3.0 |
| <= 1.2 | 2.5 |
| <= 1.6 | 2.0 |
| <= 2.0 | 1.0 |
| > 2.0 | 0.0 |

### 3.3 保底规则

当绝对估值分数为满分（6分）时，PEG 分数最低给 1.0 分。

**数据流**：`SQLite -> pe_ttm + financial_reports.profit_cagr -> calculate_long_term_peg_score()`

---

## 4. DCF 安全边际（4 分）—— 定量

### 4.1 DCF 模型参数

- **折现率**：8%
- **永续增长率**：3%
- **增长序列**：根据 profit_cagr 自动选择（5档递减序列）

### 4.2 增长序列选择

| profit_cagr | 5年增长序列 | 描述 |
|-------------|------------|------|
| < 0% | [0%, 0%, 0%, 0%, 0%] | 零增长 |
| <= 5% | [5%, 5%, 5%, 5%, 5%] | 低增长 |
| <= 10% | [8%, 6%, 5%, 5%, 5%] | 温和增长 |
| <= 15% | [15%, 12%, 10%, 8%, 8%] | 中速增长 |
| <= 20% | [20%, 15%, 12%, 10%, 8%] | 较快增长 |
| <= 25% | [25%, 20%, 15%, 12%, 8%] | 高增长 |
| <= 30% | [30%, 25%, 20%, 15%, 10%] | 极高增长 |
| > 30% | [30%, 25%, 20%, 15%, 10%] | 封顶序列 |

### 4.3 DCF 计算

```
企业价值 = Sum(未来FCF现值) + 终值现值

未来FCF现值：各年增长后的 FCF 按 8% 折现
终值 = 第5年FCF * (1+3%) / (8% - 3%)
终值现值 = 终值 / (1+8%)^5
```

### 4.4 安全边际评分

`近似市值 = PE_ttm * 最近年度净利润`

`safety_margin = DCF企业价值 / 近似市值`

| safety_margin | 得分 |
|---------------|------|
| >= 1.5 | 4.0 |
| >= 1.3 | 3.0 |
| >= 1.1 | 2.0 |
| >= 1.0 | 1.0 |
| < 1.0 | 0.0 |

**数据流**：`SQLite -> fcf + net_profit + pe_ttm -> calculate_dcf_valuation_total() -> calculate_dcf_safety_margin_score()`

---

## 5. 增长确定性（3 分）—— 定性（复用缓存）

**不独立调用 LLM**，复用 `business_model` 定性结果中的 `growth_certainty` 字段。

评估因素与 business_model 模块完全一致：
- 历史增长质量
- 行业增长前景
- 竞争地位
- 增长驱动因素

锚点：
- 3分（高确定性）、2分（中等）、1分（低）、0分（不确定）
